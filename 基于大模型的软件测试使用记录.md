# 《基于大模型的软件测试使用记录》

## 1. 使用目标

使用大模型辅助完成：
1. 测试计划撰写；
2. 黑盒/白盒用例生成；
3. 覆盖率缺口分析与补测；
4. 测试报告汇总。

## 2. 工具与环境

- 大模型代理：Cursor 内 AI Coding Agent
- 测试框架：pytest
- 覆盖率：pytest-cov
- 被测项目：minidb

## 3. 关键提示词与反馈（可截图）

## 3.1 需求到测试点

- 提示词示例：
  - “基于 minidb 的 API 与 README，提取不少于10个功能点，并标注核心流程。”
- 反馈摘要：
  - 生成了 FR-01~FR-14 需求点，包含注册、建表升级、增删改查、转换器、缓存与异常处理。

## 3.2 黑盒用例生成

- 提示词示例：
  - “按等价类、边界值、决策表方法，为 save_or_update/query/get 生成测试用例。”
- 反馈摘要：
  - 给出了合法/非法类、0/1/多条边界、id分支决策表用例建议。

## 3.3 白盒补测

- 提示词示例：
  - “根据覆盖率缺失行，给出最小增量测试，覆盖 Store.__exit__、Operation.argtosql、pformat 分支。”
- 反馈摘要：
  - 给出新增测试点并成功将覆盖率从约87%提升至95%。

## 3.4 报告生成

- 提示词示例：
  - “汇总执行结果、覆盖率、白盒方法特点、未来扩展适配方案，生成测试报告草稿。”
- 反馈摘要：
  - 形成结构化报告，含结论与成员贡献比例模板。



## 4. 使用效果评估

1. 优点：测试点覆盖系统性更强，补测效率高。
2. 风险：可能出现不贴合项目实际的建议，需人工审查。
3. 结论：在人机协作流程下，大模型显著提升测试设计与文档交付效率。
