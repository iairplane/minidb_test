# 《需求说明文档》——Minidb 对象存储项目

## 1. 项目概述

- 项目名称：`minidb`
- 项目类型：基于 SQLite 的轻量对象持久化库
- 被测源码：`/home/lzr/softwareTest/minidb/minidb.py`
- 源码规模：1008 行（已满足不少于 1000 行要求）

## 2. 目标用户与使用场景

1. 需要快速本地持久化 Python 对象的开发者。
2. 课程实验/小型工具开发者，需低学习成本 ORM 能力。
3. 希望通过表达式构造 SQL 查询的测试与教学场景。

## 3. 核心流程

**核心流程：模型注册 -> 对象保存 -> 条件查询 -> 更新 -> 删除**

1. 定义 `Model` 子类并声明字段。
2. 调用 `Store.register(Model)` 注册模型并自动建表。
3. 创建对象并 `save()` 写入数据库。
4. 使用 `query/load/get` 按条件查询。
5. 修改对象并再次 `save()` 触发更新。
6. 调用 `delete()` 或 `delete_where()` 删除数据。

## 4. 功能需求（可测试功能点，>=10）

以下编号同时作为后续测试计划中的需求追踪 ID（FR-xx）。

1. **FR-01 模型注册**：仅 `Model` 子类可注册；重复注册报错。
2. **FR-02 自动建表/升级**：首次注册自动建表；升级时可增列；类型冲突报错。
3. **FR-03 对象保存**：`save()` 首次保存后应生成主键 `id`。
4. **FR-04 对象更新**：对象已有 `id` 时再次 `save()` 执行更新。
5. **FR-05 对象删除**：`delete()` 成功后对象 `id` 置空。
6. **FR-06 条件删除**：`delete_where()` 支持表达式与 lambda 条件。
7. **FR-07 查询能力**：支持 `select/where/order_by/group_by/limit`。
8. **FR-08 聚合与函数**：支持 `count/sum/max/min/distinct` 等。
9. **FR-09 类型转换**：支持 `int/float/bool/bytes/str` 与 `JSON` 互转。
10. **FR-10 日期时间转换**：支持 `datetime/date/time` 双向转换。
11. **FR-11 自定义转换器**：通过 `@converter_for(Type)` 注册并生效。
12. **FR-12 缓存一致性**：同一主键对象重复加载应命中缓存。
13. **FR-13 异常处理**：未注册模型、查询结果非唯一等场景应抛出对应异常。
14. **FR-14 格式化输出**：`pformat/pprint/pquery` 能正确展示查询结果。

## 5. 非功能需求

1. 线程安全：`Store` 内部通过 `RLock` 支持并发访问。
2. 可维护性：接口简洁，便于补充测试与功能扩展。
3. 可观测性：支持 debug SQL 日志输出。

## 6. 约束与假设

1. 默认主键固定为 `id:int`。
2. SQLite 数据库作为唯一存储引擎。
3. 字段声明以类属性（小写）方式完成。

## 7. 验收标准

1. 黑盒与白盒测试均覆盖核心流程。
2. 覆盖率不低于 95%。
3. 测试报告能映射到上述 FR 条目。
